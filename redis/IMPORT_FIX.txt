**SYMPTOM:**
When clicking "Generate NEXUS TV Page", error dialog appeared:
"Feature Unavailable - Page generator not available"

**ROOT CAUSE:**
The import statement at the top of M3U_MATRIX_PRO.py was failing:
```python
try:
    from page_generator import NexusTVPageGenerator
    PAGE_GENERATOR_AVAILABLE = True
except ImportError:
    PAGE_GENERATOR_AVAILABLE = False  # âŒ Always False!
```

**WHY IT FAILED:**
- Both M3U_MATRIX_PRO.py and page_generator.py are in the src/ directory
- When running M3U_MATRIX_PRO.py, Python's import system looks in sys.path
- The src/ directory was NOT in sys.path by default
- Result: Python couldn't find "page_generator" module â†’ ImportError
- PAGE_GENERATOR_AVAILABLE stayed False â†’ feature disabled

This was NOT caused by recent performance improvements - it was a pre-existing
path configuration issue that surfaced during testing.

4. Import succeeds â†’ PAGE_GENERATOR_AVAILABLE = True

- Uses Path(__file__).parent.resolve() for portability
- Checks if directory already in sys.path before adding (no duplicates)
- Inserts at index 0 for priority (local modules before system modules)
- Works regardless of how the script is launched
- Compatible with both Python 3.10 and 3.11

**Alternative approaches considered:**
- Relative imports (from .page_generator import...) â†’ Requires packaging
- PYTHONPATH environment variable â†’ User configuration required
- Moving files to site-packages â†’ Too invasive

**Why sys.path.insert(0, ...) was chosen:**
- Simple, clean, and portable
- No external configuration needed
- Works immediately
- Standard Python practice for local modules
